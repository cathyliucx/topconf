# TopConf Preventive Rules

Historical fixes are encoded here as design-time and release-time requirements. They are not runtime repair loops.

| Historical failure | Underlying cause | Preventive rule | Implementation location | Enforcing test | Release gate |
| --- | --- | --- | --- | --- | --- |
| ccfddl array-root YAML failed to parse. | Parser assumed a single dictionary root. | Parser must accept both array-root and dictionary-root YAML and assign deterministic stable IDs. | `TopConf/Data/Remote/ConferenceYAMLParser.swift` | `TopConfTests/Data/Remote/ConferenceYAMLParserTests.swift` | `parser`, `catalog` |
| Supported conference categories were incomplete. | Remote loader did not explicitly cover all four required directories. | AI, CG, HI, and MX directories must be loaded by explicit source policy. | `TopConf/Data/Remote/GitHubConferenceSource.swift` | `GitHubConferenceSourceTests` | `catalog` |
| KDD/DB data could be misclassified. | Unsupported categories were treated as fallback categories. | Unsupported categories must be excluded and reported; never fallback to Emerging. | `ConferenceCategoryMapper.swift` | `testKDDDatabaseCategoryIsUnsupportedAndNotEmerging` | `parser`, `catalog` |
| Unsupported or missing ranks leaked into UI as Unranked. | Data-layer unknown state was exposed as a user filter. | UI rank filters are only A, B, C; parser still safely handles unknown/missing ranks. | `ConferenceManagementViewModel.swift`, `ConferenceYAMLParser.swift` | `ConferenceManagementUITests`, `ConferenceYAMLParserTests` | `ui`, `parser` |
| Timezone fallback could silently corrupt deadlines. | Unknown timezone strings were tolerated. | Invalid timezone identifiers must throw `unsupportedTimeZone`; AoE/UTC/PT/IANA are explicit. | `ConferenceYAMLParser.swift` | `ConferenceYAMLParserTests` | `timezone` |
| Deadline selection depended on source order. | Edition/deadline traversal was not normalized by date semantics. | Domain deadline selection must sort valid future deadlines and preserve TBD/closed semantics. | `DomainServices.swift` | `DeadlineSelectionServiceTests` | `domain` |
| Fresh production launch showed seeded tracked items. | Fixture/seed behavior leaked into production tracking state. | Production tracked repository starts empty; seed applies only to catalog data. | `DependencyContainer.swift` | `AppCompositionTests` | `integration` |
| `graphics-acm-mm` and `hci-chi` stale rows consumed slots. | Raw tracked rows were counted without resolving against accepted catalog. | Counts use resolvable tracked conferences only. | `ConferenceManagementViewModel.swift`, `TrackedConferenceListViewModel.swift` | `ConferenceManagementViewModelTests`, `TrackedConferenceListViewModelTests` | `reconciliation`, `viewmodels` |
| Orphan tracked rows rendered as `Source unavailable`. | Missing conference IDs were converted to placeholder rows. | Tracked-list output must emit only IDs that resolve to accepted catalog conferences. | `TrackedConferenceListViewModel.swift` | `TrackedConferenceListViewModelTests` | `viewmodels` |
| Orphan rows persisted after accepted refresh. | Cleanup was not tied to successful accepted refresh lifecycle. | Complete accepted refresh reconciles and deletes absent tracked IDs before publication. | `DependencyContainer.swift`, repositories | `AppCompositionTests` | `reconciliation`, `integration` |
| Cleanup could have destroyed valid data on failed/cache/seed paths. | Refresh outcome was not part of cleanup predicate. | Cleanup never runs after failed, partial, rejected, seed-only, or cache-only paths. | `DependencyContainer.swift` | `AppCompositionTests` | `reconciliation` |
| Removed orphans left reminder state behind. | Tracked deletion did not coordinate reminder cancellation. | Orphan cleanup cancels reminder rules and pending notifications for removed IDs. | `DependencyContainer.swift`, `DeadlineNotificationService.swift` | `AppCompositionTests`, notification tests | `reminders` |
| UI published stale rows after cleanup. | View models observed stale repository state before reconciliation completed. | Reconciliation must finish before tracked view-model publication and reload after deletion. | `AppRootView.swift`, `DependencyContainer.swift` | `AppCompositionTests` | `integration` |
| Onboarding did not route to direct discovery search. | First-run flow was not aligned with management flow. | Onboarding must open conference management/discovery directly with search and filters. | `OnboardingView.swift`, `ConferenceManagementView.swift` | `ConferenceManagementUITests` | `ui` |
| App icon was not release-appropriate. | Placeholder icon reached release preparation. | App icon source must be original calendar-style vector-derived artwork with generated asset sizes. | `TopConf/Assets/AppIcon` | `TopConfTests.swift` | `assets` |
| Menu-bar showed text `TopConf`. | Status item title was used instead of a template icon. | Status item must use `MenuBarCalendar` template image, no visible title, and keep accessibility. | `MenuBarStatusItem.swift` | `LauncherPanelLayoutTests` | `launcher` |
| README replacement removed useful project context. | Release docs were written as replacement instead of additive update. | Release docs must preserve original project documentation and add release-specific sections. | `README.md` | Diff review, docs gate | `release` |
| `dist/` dirtied the release tree. | Release artifacts were not ignored. | Generated release artifacts live under ignored `dist/`; reports live under ignored `.build/reports`. | `.gitignore` | `git status --short` | `git` |
| Publishing was blocked by missing `gh`. | Environment readiness did not check publication tooling early enough. | Publication mode doctor fails if `gh` or authentication is unavailable. | `topconf-doctor.sh` | Safe-failure checks | `publication` |
